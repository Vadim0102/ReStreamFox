# ReStreamFox 🦊

Легковесный, надежный стек для мультистриминга (ретрансляции) на базе **MediaMTX**, **FFmpeg** и управляющего скрипта **Watchdog**. Стек позволяет принимать один входящий поток от OBS/vMix (через RTMP, SRT, RTSP или WebRTC) и рассылать его одновременно на несколько платформ (YouTube, Twitch, Kick) с автоматическим переключением на резервное видео или статичную картинку-заставку при обрыве связи.

---

## Возможности

- **Эффективное кодирование**: Поток пережимается кодировщиком FFmpeg ровно один раз и разделяется на множество серверов вещания с помощью оптимизированного мультиплексора `tee`.
- **Широкая поддержка протоколов**: Поддерживаются протоколы RTMP, безопасный RTMPS (обязательный для Kick и Facebook Live), а также SRT и UDP с автоматическим определением формата контейнера.
- **Резервное вещание**: Автоматический переход в режим заглушки при потере основного сигнала.
- **Поддержка картинок**: В качестве заглушки можно использовать обычные изображения (`.png`, `.jpg`, `.jpeg`, `.webp`). Скрипт генерирует видеоряд из картинки и дополняет его непрерывной цифровой тишиной (`anullsrc`) для соблюдения требований принимающих платформ к аудиопотоку.
- **Веб-панель управления**: Интерактивный интерфейс на Flask + Socket.IO для мониторинга статуса входящего потока, просмотра логов FFmpeg в реальном времени, ручного переключения режимов вещания и безопасного перезапуска.

## Быстрый старт

### 1. Настройка каналов вещания
Откройте файл `outputs/outputs.txt` и укажите адреса трансляций в формате `имя=адрес` (по одному на строку):
```text
youtube=rtmp://a.rtmp.youtube.com/live2/КЛЮЧ_ТРАНСЛЯЦИИ
kick=rtmps://fa6bc2412803.global-contribute.live-video.net:443/app/КЛЮЧ_ТРАНСЛЯЦИИ
twitch=rtmp://live.twitch.tv/app/КЛЮЧ_ТРАНСЛЯЦИИ
```

### 2. Конфигурация параметров
Скопируйте шаблон конфигурационного файла:
```bash
cp watchdog/config.example.yml config.yml
```
Отредактируйте `config.yml`, указав битрейты, параметры кодирования и пароли администратора.

### 3. Добавление заставки (Опционально)
Поместите файл видео `offline.mp4` или статичную заставку `offline.png` в папку `data/` и пропишите её в `config.yml`:
```yaml
backup:
  enabled: true
  file: "/data/offline.png"
```

### 4. Сборка и запуск
```bash
docker compose up -d --build
```
Веб-интерфейс будет доступен по адресу `http://localhost:8080`. Панель администратора находится по адресу `http://localhost:8080/admin` (для авторизации используйте пароль из `config.yml`).

## Безопасность

- **Секретные данные**: Не коммитьте в публичные репозитории файлы `config.yml` и `outputs.txt`. По умолчанию они добавлены в `.gitignore`.
- **Docker Socket**: Монтирование Docker-сокета (`/var/run/docker.sock`) в контейнер UI дает ему права администратора на хосте. Если безопасность критична, не монтируйте сокет. В этом случае UI автоматически и безопасно перейдет на fallback-механизм перезапуска через управляющие файлы Watchdog.

Полное руководство по архитектуре и параметрам конфигурации находится в [docs/DOCUMENTATION_RU.md](docs/DOCUMENTATION_RU.md).
