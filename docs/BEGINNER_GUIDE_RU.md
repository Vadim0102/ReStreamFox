# Гайд для начинающих — ReStreamFox 🦊

Это руководство поможет вам с нуля развернуть и настроить ReStreamFox на вашем сервере, обезопасить его паролем и настроить программу OBS Studio для трансляции.

---

## Требования

Для запуска проекта вам понадобятся:
1. Любой сервер (VPS) или домашний компьютер на базе Linux/Windows/macOS.
2. Установленный **Docker** и **Docker Compose**.
3. Программа **OBS Studio** (или vMix) на компьютере, с которого вы будете вести трансляцию.

---

## Шаг 1. Подготовка и клонирование проекта

Подключитесь к вашему серверу через терминал и выполните команды:

```bash
# Клонируем проект
git clone https://github.com/Vadim0102/ReStreamFox.git
cd ReStreamFox
```

---

## Шаг 2. Создание рабочих конфигураций из шаблонов

Для безопасности ваших ключей все основные файлы настроек по умолчанию скрыты в `.gitignore`. Вам необходимо скопировать их из демонстрационных шаблонов:

```bash
# 1. Копируем файл направлений вещания
cp outputs/outputs.example.txt outputs/outputs.txt

# 2. Копируем конфигурацию медиасервера MediaMTX
cp mediamtx/mediamtx.example.yml mediamtx/mediamtx.yml

# 3. Копируем общие настройки проекта и админки
cp watchdog/config.example.yml config.yml
```

---

## Шаг 3. Заполнение настроек и защита сервера

Теперь настроим безопасность системы и укажем, куда стримить.

1. **Добавление платформ вещания**:
   Откройте файл `outputs/outputs.txt`:
   ```bash
   nano outputs/outputs.txt
   ```
   Добавьте адреса ваших трансляций (по одному на строку в формате `название=адрес`):
   ```text
   youtube=rtmp://a.rtmp.youtube.com/live2/ВАШ_КЛЮЧ_YOUTUBE
   kick=rtmps://fa6bc2412803.global-contribute.live-video.net:443/app/ВАШ_КЛЮЧ_KICK
   rutube=rtmp-lb-a.dth.rutube.ru/live_push/ВАШ_КЛЮЧ_RUTUBE?sinfo=ВАШ_СИСТЕМНЫЙ_ТОКЕН
   ```

2. **Защита входящего потока (Авторизация OBS)**:
   Откройте скопированный файл `mediamtx/mediamtx.yml`:
   ```bash
   nano mediamtx/mediamtx.yml
   ```
   В самом низу файла найдите блок `paths` и укажите ваши секретные логин и пароль в полях `publishUser` и `publishPass`:
   ```yaml
   paths:
     live:
       publishUser: "streamer"
       publishPass: "secure_ingest_key_here"  # Придумайте свой сложный пароль/ключ
   ```

3. **Защита веб-панели управления (Web UI)**:
   Откройте скопированный файл `config.yml`:
   ```bash
   nano config.yml
   ```
   Установите надежные пароли для доступа к веб-интерфейсу:
   ```yaml
   ui:
     admin_password: "ВАШ_СЛОЖНЫЙ_ПАРОЛЬ_ДЛЯ_ВХОДА_В_АДМИНКУ"
     admin_token: "ВАШ_СЕКРЕТНЫЙ_ТОКЕН_API"
     secret: "СЛУЧАЙНЫЙ_НАБОР_СИМВОЛОВ_ДЛЯ_СЕССИЙ"
   ```

4. **Загрузка заглушки (Опционально)**:
   Если вы хотите, чтобы при отсутствии вашего стрима показывалась заставка, положите картинку (например, `offline.png`) или видео (`offline.mp4`) в папку `data/`. В файле `config.yml` укажите путь к ней:
   ```yaml
   backup:
     enabled: true
     file: "/data/offline.png"  # Если загружена статичная картинка
   ```

---

## Шаг 4. Запуск проекта

Запустите контейнеры в фоновом режиме:
```bash
docker compose up -d --build
```
Убедиться, что все службы успешно запустились, можно командой: `docker compose ps`.

---

## Шаг 5. Настройка вещания в OBS Studio

Откройте OBS Studio на вашем игровом/рабочем компьютере, перейдите в **Настройки** -> **Трансляция**. Выберите один из способов подключения:

### Способ 1. Через протокол SRT (Рекомендуется)
*Обеспечивает высокую устойчивость соединения и минимальную задержку.*

1. В поле **Сервис** выберите **Настраиваемый** (Custom).
2. В поле **Сервер** укажите единую строку подключения, содержащую ваши имя пользователя и пароль из настроек `mediamtx.yml`:
   ```text
   srt://<IP_ИЛИ_ДОМЕН_ВАШЕГО_СЕРВЕРА>:8890?streamid=publish:live:streamer:secure_ingest_key_here
   ```
   *(Замените `streamer` и `secure_ingest_key_here` на логин и пароль, заданные вами в `mediamtx.yml`)*
3. Поле **Ключ потока** оставьте **пустым**.
4. Нажмите **Применить** и запустите трансляцию.

### Способ 2. Через протокол RTMP (Альтернативный)
1. В поле **Сервис** выберите **Настраиваемый** (Custom).
2. В поле **Сервер** укажите:
   ```text
   rtmp://<IP_ИЛИ_ДОМЕН_ВАШЕГО_СЕРВЕРА>:1935/live?user=streamer&pass=secure_ingest_key_here
   ```
3. Поле **Ключ потока** оставьте **пустым** или введите любое случайное слово.
4. Нажмите **Применить** и запустите трансляцию.

---

## Шаг 6. Мониторинг вещания

1. Перейдите в браузере по адресу `http://<IP_ВАШЕГО_СЕРВЕРА>:8080`.
2. Вы увидите публичную страницу мониторинга.
3. Для доступа к панели управления нажмите **Show logs** или перейдите по адресу `/admin`. Система потребует авторизацию — введите пароль, заданный вами в `config.yml` в поле `ui.admin_password`.
4. В админ-панели логи FFmpeg будут транслироваться в реальном времени. При сбоях интернета или перезапуске OBS система автоматически удержит трансляции на платформах с помощью вашей заглушки.
