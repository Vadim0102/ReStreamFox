# Contributing

Thanks for wanting to contribute. Short guidelines:

- Fork the repo and create a feature branch.
- Keep changes focused and create small PRs.
- Add tests or at least manual verification steps for behavioral changes.
- Update README(s) and add examples for new features.
- Use `docker compose build` to test image builds locally.

If your change affects `config.yml` shape, update the schema in `watchdog/watchdog.py`.
